# ObsCodec 项目概述

## 研究问题

如何在多智能体协同场景下，对观测数据进行高效压缩编码，使得在带宽受限、信道不可靠的条件下仍能保持任务性能？传统编解码器追求重建精度，而语义通信要求编码在"对下游任务有用"和"传输稳健"之间取得平衡。

## 方法

本项目将五类编解码器——PCA（线性）、标准AE（非线性重建）、标量量化（数字基线）、β-VAE（概率语义瓶颈）和VQ-VAE（离散码本）——统一框架下比较，覆盖**7个MPE场景**（18-90维，3-15个智能体）。

**扩展基准**聚焦于高维扩展与坍塌预防：β-VAE在90维观测上的后验坍塌通过free-bits机制彻底解决（FB=0.1通用抗坍塌，最小有效剂量0.02 nats/dim），VQ-VAE在6种信道损伤模型下完成跨场景评估，统一编解码器验证了跨场景正迁移。

## 核心发现

1. **FB=0.1在所有场景和智能体数量（3→15）下通用预防后验坍塌**——坍塌率90%→0%
2. **最小有效FB剂量 = 0.02 nats/dim**——比文献值（0.5-2.0）低25-100倍
3. **KL与维度无关**——18-90维范围恒定于~1.5 nats，MSE改善35-39%
4. **单独扩大解码器对抗坍塌无效**——瓶颈在率项，不在解码器容量
5. **AWGN中等SNR改善VQ-VAE MSE**——去噪正则化效应
6. **统一编解码器优于单场景模型**——在最难任务上MSE降低5.0%
7. **跨场景信息结构差异**比原始维度对坍塌行为影响更大

## 技术栈

Python 3.10+, PyTorch 2.6.0, CUDA 12.6, RTX 3050 (8 GB)

## 规模

**263个训练模型，15个结果JSON，17张图表，13个数据集**（33,333样本/集），15个实验脚本（含4个语义通信脚本）

## Phase 3 完成

语义通信实验设计已完成：可微信道层（AWGN重参数化、丢包直通估计）、联合信源信道编码（JSCC-BetaVAE、JSCC-VQ-VAE）、任务感知压缩损失（self_only、weighted MSE）、端到端闭环原型（obs→编码→信道→解码→策略→任务）。库新增：`channel/diff_channel.py`、`models/jscc.py`、`task_metrics.py`。
