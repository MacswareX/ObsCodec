## 状态分类

| 状态 | KL范围 | 含义 |
|------|--------|------|
| 正常 (OK) | KL ≥ 0.10 | 后验携带信息；编解码器正常工作 |
| 低信息 (LOW) | 0.01 ≤ KL < 0.10 | 部分坍塌；信息量边缘 |
| 坍塌 (COLLAPSED) | KL < 0.01 | 后验等于先验；潜变量无观测信息 |

## 解读说明

- **Free-bits抗坍塌**：FB=0.1在所有场景和智能体数量下通用预防坍塌。最小有效剂量(FB=0.02)比文献默认值低25-100倍，表明坍塌阈值远低于此前假设。
- **KL维度无关性**：FB=0.1下，绝对KL在18-90维观测范围内恒定于~1.5 nats。逐维度free-bits地板意味着总KL仅取决于超过地板的维度数量——在规模变化时保持不变。
- **扩大解码器无效**：将解码器隐藏维度从编码器的1×扫描到4×显示零抗坍塌效果。坍塌瓶颈在率(KL)项，而非解码器容量。
- **VQ-VAE信道效应**：中等SNR的AWGN对VQ-VAE起到隐式去噪正则化作用。瑞利衰落更具破坏性，尤其在低SNR下。
- **跨场景迁移**：统一编解码器在最难场景(spread_xhd)上的优越表现表明，共享表示通过来自简单场景的正则化使高维任务受益。
- 所有β-VAE结果使用修正的free-bits逻辑：`max(0, KL_per_dim.mean(dim=0) - free_bits).sum()`——逐维度批次平均截断，而非逐样本。
- 重建MSE是代理指标。Phase 3脚本（7-10）实现了任务感知、JSCC和端到端闭环评估——以下记录实验设计，待规模化执行。
- 所有β-VAE结果使用修正的free-bits逻辑：`max(0, KL_per_dim.mean(dim=0) - free_bits).sum()`——逐维度批次平均截断，而非逐样本。

## Phase 3：语义通信 — 实验设计

脚本7-10实现了四个语义通信实验。以下记录实验设计、测量指标和预期结果（脚本已就绪，完整执行需GPU时间）。

### 表9：可微信道基准（`7_diff_channel.py`）

| 训练信道 | 测试条件 | 编解码器 | β | FB | 测量指标 |
|---------|---------|--------|---|---|---------|
| 无（基线） | 干净 | BetaVAE | 2.0 | 0.1 | MSE, KL |
| DiffAWGN 20dB | 干净、匹配20dB、非匹配10/5/0dB | JSCC-BetaVAE | 2.0 | 0.1 | 各测试SNR下MSE |
| DiffAWGN 10dB | 干净、匹配10dB、非匹配20/5/0dB | JSCC-BetaVAE | 2.0 | 0.1 | 各测试SNR下MSE |
| DiffAWGN 5dB | 干净、匹配5dB、非匹配20/10/0dB | JSCC-BetaVAE | 2.0 | 0.1 | 各测试SNR下MSE |
| DiffAWGN 0dB | 干净、匹配0dB、非匹配20/10/5dB | JSCC-BetaVAE | 2.0 | 0.1 | 各测试SNR下MSE |
| DiffErasure 10% | 干净、匹配10%、非匹配30% | JSCC-BetaVAE | 2.0 | 0.1 | 各丢包率下MSE |
| DiffErasure 30% | 干净、匹配30%、非匹配10% | JSCC-BetaVAE | 2.0 | 0.1 | 各丢包率下MSE |

测试核心JSCC假设：可微信道训练是否提升匹配和非匹配信道条件下的鲁棒性。重参数化技巧使梯度可流过AWGN；直通估计器处理丢包信道。

### 表10：JSCC训练网格（`8_jscc_training.py`）

| 维度 | 取值 |
|------|------|
| 场景 | simple_spread (30维), spread_hd (48维), spread_xhd (90维) |
| 编解码器 | BetaVAE β=0.1, BetaVAE β=2.0, VQ-VAE (cb=512) |
| Free Bits | 0.0, 0.1 |
| 训练信道 | 干净, AWGN (20/10/0 dB), 丢包 (10%, 30%) |
| 测试信道 | 干净, AWGN (20/10/0/-5 dB), 丢包 (0/10/30%) |
| 每配置指标 | MSE, KL, NMSE（按数据方差归一化） |

全因子设计：3场景 × 3编解码器 × 2 FB × 6训练信道 × 8测试信道 = 最多864个数据点。核心JSCC实验，测试信道内环训练能否跨编解码器族和信道类型泛化。

### 表11：任务感知损失（`9_task_aware.py`）

| 参数 | 取值 |
|------|------|
| β | 0.01, 0.1, 1.0 |
| 任务损失类型 | none（基线）、self_only（前2维MSE）、weighted（0.7×自身 + 0.3×他人） |
| 任务权重 | 0.01, 0.1, 0.5, 1.0 |
| 场景 | simple_spread, simple_tag, simple_world_comm |

**核心问题**：任务感知损失能否替代或补充free-bits以维持潜变量活性？测量指标：总MSE、自身位置MSE、其他智能体MSE、协调差距（自身与其他人误差比）、KL散度、逐智能体MSE分解。

### 表12：端到端闭环（`10_end_to_end.py`）

| 条件 | 压缩 | 信道 | 描述 |
|------|------|------|------|
| no_compression | 无 | 干净 | 原始观测 → 启发式策略（性能上界） |
| jscc_clean | JSCC-BetaVAE (β=2.0, FB=0.1) | 干净 | 编码→解码→策略（仅压缩代价） |
| jscc_noisy | JSCC-BetaVAE (β=2.0, FB=0.1) | AWGN（可配置SNR） | 完整管道含信道噪声 |

每条件10个回合，每回合200步。指标：到目标最终距离（最后10步均值）、早/晚期平均距离、路径效率、碰撞次数。SpreadSimulator（5智能体+5地标）闭环连接观测→编码→信道→解码→策略→动作。
